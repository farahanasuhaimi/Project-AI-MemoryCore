## Main Work Done

### 1. win-board — Three Bug Fixes

**Bug: Good To Do tasks not carrying forward**
- Root cause: carry-forward query in `DashboardController.php` had `whereIn('section', ['must', 'should', 'park'])` — `good` was intentionally excluded in an earlier design but Nufa wanted it included.
- Fix: added `'good'` to the whereIn list.

**Bug: LATE/URGENT badge not showing on carried-forward Must tasks**
- Root cause: badge logic relied on `$task->days_late` being dynamically set on Eloquent model instances via `each()`. The attribute wasn't reliably accessible in the blade view (Eloquent dynamic property persistence edge case).
- Fix: replaced with inline `@php` block in the blade that computes directly from `$task->date` (already a Carbon via model cast). Also changed `=== 1` strict comparison to `>= 1` to handle type edge cases.

**New behaviour: Undone Should tasks escalate to Must the next day**
- Not a bug — a new business rule Nufa clarified.
- Fix: one DB update in `DashboardController::index()` — promotes undone `should` tasks with `date < today` to `section = 'must'` before carry-forward runs. One-way, permanent escalation.
