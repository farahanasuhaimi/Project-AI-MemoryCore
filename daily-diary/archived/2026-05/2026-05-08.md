### Main Topics

1. **Memory System Refactor — Rudy Owns Note-Saving**
   Previously all three agents could emit `SAVE_NOTE` themselves — messy and uncoordinated. Redesigned so Rudy is the sole decision-maker for what gets saved and where. Each agent now has its own notes library (`finance_notes.json`, `marketing_notes.json`, `hr_notes.json`) plus Rudy has his own (`rudy_notes.json` for standing instructions). Agents read their notes as preloaded context; they never write them. Rudy emits `SAVE_FINANCE_NOTE`, `SAVE_MARKETING_NOTE`, `SAVE_HR_NOTE`, or `SAVE_RUDY_NOTE` tags during the routing call.

   Nufa asked if Rudy needs a memory core like Timothy has. Good question. Answer: not a full memory core — Rudy is a business coordinator, not a personal companion. He doesn't need relationship history or session recaps. What he does need is a place for standing instructions directed at him specifically — how Nufa wants him to behave, cross-domain preferences. That's `rudy_notes.json`. Minimal, purposeful.

   During testing, the marketing agent said "Save to `content_week1_thread.md`" — but nothing was actually written. It was just text in the response. Fixed by adding a `SAVE_CONTENT_FILE` tag: agent emits `<SAVE_CONTENT_FILE filename="...">full content</SAVE_CONTENT_FILE>`, router writes the file to `library/content/`. Confirmed working during testing — `content_week1_thread.md` was created.
